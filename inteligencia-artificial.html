ESTACIÓN 1

Datos, modelos y algoritmos

Estos conceptos permiten comprender cómo una IA aprende patrones, procesa información y entrega resultados útiles.

D
Datos

Los datos son ejemplos, registros o evidencias que alimentan el entrenamiento y la evaluación de modelos.

datos = ejemplos + contexto
  • calidad = exactitud y relevancia
  • cantidad = volumen de ejemplos
  • variedad = casos diferentes
Base: confiabilidad
M
Modelo

Representa el sistema que identifica patrones y genera predicciones o respuestas.

modelo = datos + aprendizaje
salida = entrada procesada

Depende del tipo de datos, el entrenamiento y la tarea que se desea resolver.

Base: desempeño
A
Algoritmo

Es el procedimiento que permite transformar datos en decisiones, predicciones o contenido.

algoritmo = pasos + reglas
resultado = proceso aplicado

Su utilidad depende de la tarea, los datos y la forma de evaluar sus resultados.

Base: lógica

ESTACIÓN 2

Prompts y contexto

Un prompt bien diseñado guía al sistema de IA mediante objetivo, contexto, rol, formato y restricciones.

💧 Prompt breve Poco contexto
→
🛢️ Prompt guiado Contexto suficiente
→
🍯 Prompt experto Rol, formato y criterios
P
Prompt

Define la instrucción que recibe la IA para producir una respuesta alineada con la tarea.

prompt = tarea + contexto + formato
  • tarea = lo que se solicita
  • contexto = información relevante
  • formato = cómo debe responder
Base: claridad
C
Contexto

Aporta datos, restricciones y criterios para que la respuesta sea más precisa y útil.

contexto = antecedentes + límites

Permite reducir ambigüedades y adaptar la respuesta al problema real.

Base: pertinencia
🔁 Iteración del prompt
📝 Primer intento

Una instrucción general puede producir respuestas incompletas, porque no define criterios ni formato.

🎯 Prompt refinado

Al agregar contexto, ejemplos y restricciones, mejora la utilidad de la respuesta y se facilita la verificación.

🎯

Idea clave: iterar el prompt mejora la calidad de la salida, pero siempre requiere revisión humana y fuentes confiables.

🗜️
ESTACIÓN 3

Generación, sesgos y verificación

Las respuestas generadas por IA pueden ser útiles, pero deben revisarse para detectar errores, sesgos o información inventada.

S
Salida

Es el texto, imagen, código o recomendación que genera la herramienta de IA.

salida = prompt + modelo
Base: utilidad
B
Sesgo

Describe una tendencia del sistema a favorecer ciertos patrones, grupos o resultados.

sesgo = datos + diseño + contexto

Puede reducirse con revisión, datos diversos y evaluación responsable.

V
Verificación

Evalúa si la salida es correcta, útil y consistente con fuentes confiables.

verificación = fuentes + criterio

Una respuesta clara no siempre es verdadera; debe comprobarse antes de aplicarse.

Base: confianza
🌡️
REVISIÓN

Mejora iterativa de respuestas

Cuando cambia el contexto, la instrucción o los datos, la respuesta de la IA puede mejorar. La revisión permite corregir errores y afinar el resultado.

mejora = respuesta + retroalimentación
⚙️
¿Dónde importa?

Reportes técnicos, planeación de clase, análisis de datos, generación de contenido y automatización de procesos académicos o industriales.

⚙️
ESTACIÓN 4

Automatización y productividad

La IA puede integrarse en flujos de trabajo para reducir tareas repetitivas, organizar información y apoyar la toma de decisiones.

F
Flujo de trabajo

Es la secuencia de pasos que conecta entradas, herramientas de IA, revisión humana y resultados.

flujo = entrada + IA + revisión

Ayuda a documentar procesos, reducir errores y repetir tareas con mayor consistencia.

Base: organización
P
Productividad

Es la capacidad de producir mejores entregables en menos tiempo sin perder revisión ni calidad.

productividad = rapidez + calidad
  • rapidez = menos tareas repetitivas
  • calidad = revisión y mejora
  • criterio = decisión humana

Evaluación diagnóstica

1. ¿Cuál de las siguientes opciones define mejor a la Inteligencia Artificial?

  1. Sistemas capaces de realizar tareas que normalmente requieren inteligencia humana.
  2. Una hoja de cálculo con fórmulas básicas.
  3. Un dispositivo que solo almacena archivos.
  4. Un programa que no utiliza datos ni reglas.

2. ¿Qué elemento es fundamental para entrenar o ajustar muchos sistemas de IA?

  1. Datos
  2. Pintura
  3. Voltaje doméstico
  4. Velocidad del viento

3. El aprendizaje automático se refiere principalmente a:

  1. Sistemas que mejoran su desempeño a partir de datos.
  2. Copiar texto sin analizarlo.
  3. Apagar una computadora automáticamente.
  4. Diseñar únicamente piezas mecánicas.

4. ¿Cuál es un ejemplo de IA generativa?

  1. Crear texto, imágenes o código a partir de instrucciones.
  2. Medir una distancia con una regla.
  3. Guardar archivos en una carpeta.
  4. Imprimir un documento sin cambios.

5. Un prompt es:

  1. La instrucción o entrada que se entrega a un sistema de IA.
  2. Un componente físico de una computadora.
  3. Un error de conexión a internet.
  4. Un tipo de cable eléctrico.

6. La ingeniería de prompts busca principalmente:

  1. Diseñar instrucciones claras para obtener mejores respuestas.
  2. Aumentar el brillo de la pantalla.
  3. Reemplazar todas las evaluaciones humanas.
  4. Eliminar la necesidad de verificar información.

7. ¿Qué significa que una IA pueda presentar una “alucinación”?

  1. Que genere información incorrecta o inventada con apariencia convincente.
  2. Que deje de consumir energía.
  3. Que funcione sin datos.
  4. Que solo responda con números.

8. Un modelo de lenguaje grande se utiliza principalmente para:

  1. Procesar y generar lenguaje natural.
  2. Medir temperatura ambiente.
  3. Soldar componentes electrónicos.
  4. Controlar directamente una válvula hidráulica.

9. ¿Cuál es una buena práctica al usar IA en trabajos académicos?

  1. Verificar la información y citar las fuentes utilizadas.
  2. Copiar todo sin revisar.
  3. Ocultar el uso de herramientas cuando se exige transparencia.
  4. Aceptar cualquier respuesta como verdadera.

10. ¿Qué es un sesgo en IA?

  1. Una tendencia del sistema a favorecer ciertos patrones o resultados.
  2. Una mejora automática garantizada.
  3. Un formato de imagen.
  4. Una contraseña del sistema.

11. En IA, un conjunto de datos sirve para:

  1. Entrenar, probar o validar modelos.
  2. Reemplazar completamente el criterio humano.
  3. Eliminar todos los errores de un sistema.
  4. Convertir texto en electricidad.

12. ¿Cuál es una tarea común de clasificación con IA?

  1. Separar correos en deseados y no deseados.
  2. Calcular el área de un círculo sin datos.
  3. Apretar un tornillo manualmente.
  4. Encender una lámpara con un interruptor simple.

13. La automatización con IA puede ayudar a:

  1. Reducir tareas repetitivas y apoyar decisiones.
  2. Eliminar todas las responsabilidades profesionales.
  3. Garantizar resultados perfectos sin revisión.
  4. Evitar el uso de datos.

14. ¿Qué opción describe mejor el uso responsable de IA?

  1. Usarla con verificación, privacidad, transparencia y criterio humano.
  2. Usarla para sustituir toda revisión.
  3. Compartir datos sensibles sin permiso.
  4. Aceptar cualquier salida sin análisis.

15. ¿Qué es una API en el contexto de herramientas de IA?

  1. Una interfaz para conectar aplicaciones con servicios de IA.
  2. Un archivo de imagen sin compresión.
  3. Un tipo de examen presencial.
  4. Una unidad de memoria física.

16. ¿Qué conviene incluir en un buen prompt?

  1. Objetivo, contexto, formato esperado y restricciones.
  2. Solo una palabra aislada.
  3. Datos personales innecesarios.
  4. Instrucciones contradictorias sin prioridad.

17. Un chatbot con IA se caracteriza por:

  1. Interactuar mediante lenguaje natural con el usuario.
  2. Funcionar únicamente como calculadora básica.
  3. Requerir siempre impresión en papel.
  4. No poder procesar texto.

18. La privacidad de datos en IA es importante porque:

  1. Evita exponer información personal o sensible sin autorización.
  2. Impide aprender cualquier concepto técnico.
  3. Solo aplica a imágenes decorativas.
  4. No tiene relación con herramientas digitales.

19. ¿Qué resultado se espera de un proyecto integrador con IA?

  1. Una solución aplicada que resuelva o mejore un proceso real.
  2. Una lista sin propósito de herramientas populares.
  3. Una copia exacta de un ejemplo sin adaptación.
  4. Un documento sin evidencia ni prueba.

20. ¿Cuál es una limitación de las herramientas de IA actuales?

  1. Pueden equivocarse y requieren supervisión humana.
  2. Siempre conocen información actualizada sin conexión.
  3. Nunca presentan sesgos.
  4. No necesitan instrucciones claras.
